Real or Fake?
Detecting the legitimacy of electronically-submitted bank notes with Machine Learning

A data science project by an AI Camp instuctor-in-training

See my work below

What does the project do?

I utilize a dataset of 1372 preprocessed bank note images, with each image's variance, skew, curt, and entropy recorded. Furthermore, each bank note was classified to be either real or fake. My goal was to use this dataset to predict the legitimacy of bank notes, given the required features are known.

Exploration


Exploration of data: Correlation table

First, we created a correlation table to see how different features correlated with one another. Note that image curt and entropy seem to be the most positively correlated, while image skew and curt seem to be the most negatively correlated.



Exploration of data: Scatterplots

By looking at scatterplots, we can visually see trends in the data. Notice that the plot of image curt vs skew seems to be negatively correlated, while the plot of entropy vs skew seems to have a parabolic nature. We will examine these plots closer below.



Scatterplots of identified correlated features

Through the exploration above, we have identified the features in the following scatterplots to be the most correlated. Thus, we decided to replot them with labels depicting their respective class. Machine Learning


Gradient Boosting: Explanation

Gradient boosting is a machine learning method in which a decision tree is built to classify points in the data. In gradient boosting, we can use multiple decision trees when we fit the model to the data. As data is passed through the tree to generate a prediction, it is passed through each node until a leaf is reached, which represents its classofication. In this case, a 0 or 1 representing real or fake.

Gradient Boosting: Results

Here we have a confusion matrix with the results of gradient boosting applied on our testing dataset. A confusion matrix for a classification problem tells us the number of correct and incorrect predictions for every class. In this case, the model classified only one real bank note as a fake note. Gradient Boosting performed very well on this dataset.

By calculating the total correct classifications over all examples classified, we get an average accuracy of around 99%.

Neural Networks: Explanation

Neural networks are one of the most powerful machine learning approaches for data with complicated patterns, but are very prone to overfitting to simple data. Thus, when training with neural networks, it is very important to set aside a partition of dataa to use to verify the accuracy of the model

Neutral Networks: Results

Looking at the confusion matrix below, the neural network performed well.

In training any neural network, we have the option of using an "early stopping" token which still stop iterating on the dataset when the accuracy increase becomes minimal. Since our dataset was only several hundred entries, we decided not to implement early stopping. Early stopping is useful when we have very large datasets, or expect fitting the data to be very difficult. In total, the NN performed at 100% accuracy in classification.

Conclusion


In conclusion, the neural networks marginally performed better than gradient boosting on our dataset for detecting real and fake bank notes. Because the data had only a few features, each one of them was vital to achieving the accuracy the models did.

So what?

So what is the significance of our project? Now that we have a model that can accurately detect a real or falsified bank note, banks can implement this model into their business. For example, when a user uploads a bank note image our model may classify as fake, they can prompt the user to upload another photo or perform another method of verification.

Possible Extensions

How can we improve our model? Our model achieved 100% accuracy, meaning utilizing other means of learning may not be beneficial to us. Rather, having a larger dataset would be a great way to ensure that our model is robust and applicable to some 'general' bank note instance. Furthermore, what if we don't have more data available? We could use an ensemble method to combine BOTH of the models we have trained. This may help us increase generality. To extend this project moving foward, I would suggest that students create a spot for users to upload a photo of a bank note, to which our site will return whether the model thinks its a real or fake note. To do this, we would first have to add a spot for users to upload images, and then extract all the necessary features from the image before leting the model make a prediction based on the data.
